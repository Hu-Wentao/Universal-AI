## **一、核心概念拆解**

### **1\. Universal App（通用应用）**

指不局限于单条公链的应用，能在多条公链（如 Ethereum、Bitcoin、Solana）上无缝运行，用户无需切换链环境就能使用功能。

-   核心特点：跨链兼容性强，资产和数据可在不同链间自由流转，无需为每条链单独开发版本。
    

### **2\. Universal EVM（通用以太坊虚拟机）**

是 ZetaChain 内置的兼容 EVM 的执行环境，也是通用应用的核心运行载体。

-   核心价值：解决了不同公链虚拟机不兼容的问题，开发者用 Solidity 等 EVM 原生语言写的合约，无需修改就能跨链执行，无需适配 Bitcoin（非 EVM）、Solana（独立虚拟机）等不同链的技术标准。
    

### **3\. Omnichain Smart Contract（全链智能合约）**

部署在 Universal EVM 上的合约，能直接与多条 Connected Chains（连接链）交互，而非在每条链上单独部署副本。

-   核心能力：可跨链读取数据、转移资产、触发其他链上的合约逻辑，实现 “一次部署，全链可用”。
    

### **4\. Gateway（网关）**

ZetaChain 与各公链的 “连接桥梁”，是跨链交互的关键组件，每条 Connected Chain 都对应一个专属 Gateway。

-   核心功能：接收并验证对应链上的交易 / 消息，将其转发到 ZetaChain 的 Universal EVM；同时将 ZetaChain 的跨链指令（如资产转移、合约调用）同步到目标链，确保跨链操作的安全性和一致性。
    

### **5\. Connected Chains（连接链）**

指与 ZetaChain 通过 Gateway 建立连接的公链，涵盖 EVM 链（Ethereum、BSC）、非 EVM 链（Bitcoin、Dogecoin）、独立虚拟机链（Solana、Aptos）等。

-   核心意义：扩大了通用应用的覆盖范围，让不同链上的用户、资产、数据能互通，避免 “链上孤岛”，提升整个区块链生态的流动性和可用性。

## **三、跨链工作流程（以 “ETH 链资产转移到 BTC 链” 为例）**

1.  用户在 Ethereum 链发起交易：将 ETH 资产转入 ETH 对应的 Gateway，并附带目标链（BTC 链）、接收地址等跨链指令。
    
2.  Gateway (ETH) 验证交易：确认用户资产到账后，生成跨链消息并签名，发送到 ZetaChain 的 Universal EVM。
    
3.  Universal EVM 处理逻辑：触发全链合约执行，锁定用户在 Gateway 的 ETH 资产，同时生成对应的跨链凭证（记录资产金额、接收方等信息）。
    
4.  ZetaChain 同步指令到目标网关：将跨链凭证和资产转移指令转发到 Gateway (BTC)。
    
5.  Gateway (BTC) 执行跨链操作：在 Bitcoin 链上生成对应的资产（如包装 BTC），发送到用户指定的接收地址，并将执行结果反馈给 ZetaChain。
    
6.  完成跨链闭环：ZetaChain 记录整个交易流程，确保两端操作一致，用户最终在 BTC 链收到对应资产，跨链过程结束。
