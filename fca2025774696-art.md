# **ZetaChain的总结**：

**一句话总结：**ZetaChain 旨在成为一个通用的、能够原生连接所有区块链的底层平台，让开发者能够像开发单链应用一样，轻松构建跨链应用（全链应用）。

其终极目标是解决当前区块链生态的碎片化问题，成为去中心化世界的“通用入口”和“连接器”。

## **一、核心问题：为什么需要 ZetaChain？**

当前区块链生态存在以下痛点，而 ZetaChain 正是为了解决它们而生：

痛点 ZetaChain 的解决方案

孤岛效应：每条链都是一个封闭生态，资产和数据难以互通。 枢纽模型：ZetaChain 自身作为中心枢纽，统一连接和管理所有链。

开发复杂：构建跨链应用需要在多条链上部署合约，并维护复杂的跨链通信逻辑。 通用应用：开发者只需在 ZetaChain 上部署一次智能合约，即可管理所有连接链的交互。

用户体验差：用户需要手动使用跨链桥，操作繁琐，成本高，风险大。 无缝体验：用户在一个链上的单一操作，即可由通用应用自动编排复杂的多链交易。

资产形式不统一：跨链后产生大量的封装资产，流动性分散，安全性依赖第三方。 原生资产转移：通过 ZRC-20 标准，实现资产在不同链间的原生转移，无需封装。

## **二、核心技术架构与组件总结**

ZetaChain 通过一套精密的组件协同工作，实现上述愿景。

-   **通用应用**
    

\- 定义：部署在 ZetaChain 上的智能合约，但其能力不局限于 ZetaChain 本身。

\- 核心能力：

\- 读取任何连接链的状态。

\- 向任何连接链发送资产和触发合约调用。

\- 编排涉及多条链的复杂交易流程。

-   **关键技术支柱**
    

1.  通用权益证明：基于 Cosmos SDK，提供网络安全和快速最终性。
    
2.  通用 EVM：完全兼容以太坊 EVM，开发者可使用 Solidity 和熟悉的所有工具。
    
3.  Omnichain 智能合约：具备原生跨链能力的合约的执行环境。
    

-   **关键协议与组件**
    

\- CCTX：跨链交易的全局记录。每一个跨链操作都会生成一个 CCTX 来追踪其状态（待处理、已确认、已回退等），是保证操作可追溯性的核心。

\- Gateway（网关）：用户与 ZetaChain 交互的统一入口。无论是存入资产还是调用合约，用户都通过连接链上的网关合约进行。

\- 入站：从外部链存入资产或调用 ZetaChain 合约。

\- 出站：从 ZetaChain 提取资产或调用外部链合约。

\- ZRC-20：ZetaChain 上的代币标准，用于代表所有连接链上的原生资产（如 ETH 在 ZetaChain 上就是 ZRC-20 形式的代币）。它是实现资产跨链的桥梁。

-   **Gas 费机制**
    

\- 在 ZetaChain 上执行：用户从连接链调用通用应用时，在 ZetaChain 端的合约执行是 Gasless（无Gas费） 的，用户只需支付源链的 Gas。

\- 从 ZetaChain 提款/调用：通用应用需要支付目标链的 Gas，这笔费用以目标链 Gas Token 的 ZRC-20 形式支付。

# ZetaChain 本地开发环境的全过程

## 第一阶段：安装核心前置依赖

**1\. 安装 Node.js 和 npm**

目的：这是运行 ZetaChain CLI 和大多数 JavaScript/TypeScript 工具链的基础。

操作：

访问 Node.js 官网。

强烈建议下载并安装 v21.x 或更高版本的 LTS（长期支持）版本。这将同时安装 Node.js 和其包管理器 npm。

打开终端（Terminal 或 Command Prompt），运行以下命令验证安装是否成功：

node --version # 应输出 v21.x.x 或更高 npm --version # 应输出对应的 npm 版本号

**2\. 安装 Git**

目的：用于克隆模板项目和进行版本控制。

操作：

访问 Git 官网下载并安装。

安装后，在终端中验证：

git --version

**3\. 安装 jq**

目的：一个轻量级的命令行 JSON 处理器，用于解析脚本输出（尤其在 Localnet 中）。

操作：

macOS： 使用 Homebrew 安装是最简单的方式。

brew install jq

Linux (Ubuntu/Debian)：

sudo apt-get update && sudo apt-get install jq

Windows： 建议通过 WSL 安装 Linux 发行版，然后在 WSL 内使用上述 Linux 命令安装。或者，可以从 官网 下载 exe 文件并配置环境变量。

验证：

jq --version

**4\. 安装 Foundry**

目的：一个快速的以太坊开发工具包。ZetaChain 的 Localnet 依赖于它来运行本地 EVM 环境。

操作：

打开终端，运行以下命令（适用于 macOS 和 Linux）：

curl -L [https://foundry.paradigm.xyz](https://foundry.paradigm.xyz) | bash

安装脚本会输出提示。完成后，重启你的终端或运行 source ~/.bashrc（或 ~/.zshrc，取决于你使用的 Shell）来加载新路径。

然后，运行以下命令来安装 Foundry 工具：

foundryup

验证：安装完成后，检查 forge 命令是否可用。

forge --version

## 第二阶段：安装 ZetaChain CLI 并验证

**1\. 安装 CLI**

目的：这是与 ZetaChain 交互的最主要工具，用于管理账户、发起跨链调用、查询状态等。

操作：

使用 npm 全局安装：

npm install -g @zetachain/zetachain-cli

注意：官方文档的命令是 npm install -g zetachain，但当前更推荐使用 @zetachain/zetachain-cli 包。

**2\. 验证 CLI 安装**

操作：运行以下命令，它会列出所有 ZetaChain 连接的链。如果成功显示一个很长的链列表（如 Ethereum, BSC, Polygon 等），说明你的 CLI 安装正确且能访问网络信息。

zetachain query chains list

## 第三阶段：启动并测试 Localnet（推荐）

**1\. 启动 Localnet**

目的：在本地计算机上启动一个完整的、自包含的测试环境，包含 ZetaChain、EVM 链以及（可选）Solana、Sui 和 TON。这是最快、最免费的开发和测试方式。

操作：

确保 Foundry 已安装后，只需一个命令：

zetachain localnet init

注意：首次运行会下载必要的 Docker 镜像，这可能需要一些时间。请确保你的系统已安装并运行了 Docker Desktop。

**2\. 验证 Localnet 运行状态**

操作：启动后，CLI 会输出日志。你可以使用以下命令检查核心服务的状态：

zetachain localnet status

它会显示 ZetaChain 节点、EVM 链节点等是否在正常运行
